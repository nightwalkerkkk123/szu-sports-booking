# szu-sports-booking（深大体育馆预约）

**⚠️ 安全规则 — 真实预约会占用实际名额，调用前必须用户明确确认。先 dry-run，再确认，最后执行真实预约。**

## 项目信息

- **目标 URL**: `https://ehall.szu.edu.cn/qljfwapp/sys/lwSzuCgyy/index.do#/sportVenue`
- **Python**: 3.10+

## 路径配置

项目路径通过环境变量 `SZU_BOOKING_DIR` 设置，默认为当前目录 `.`。

**首次使用需要配置**：
```bash
# Windows PowerShell
setx SZU_BOOKING_DIR "E:\szu-sports-booking"

# Linux/macOS / Git Bash / WSL
export SZU_BOOKING_DIR="/path/to/szu-sports-booking"
```

## 配置文件

| 文件 | 用途 |
|------|------|
| `configs/config.yaml` | 账号、校区、场馆、时间段等业务配置 |
| `.env` | 密码存储（按学号后4位命名，如 `SZU_PASSWORD_0090=密码`）|

## 命令格式

```bash
cd "$SZU_BOOKING_DIR" || cd .
python main.py [选项]
```

### 参数说明

| 参数 | 说明 | 必填 | 示例 |
|------|------|------|------|
| `--username` | 学号 | ✅ | `2023150090` |
| `--password` | 密码 | ✅ | `11282577` |
| `--campus` | 校区 | ✅ | `粤海校区` / `丽湖校区` |
| `--sport` | 体育项目 | ✅ | `网球` / `羽毛球` / `乒乓球` |
| `--date` | 日期索引 | ✅ | `0`=今天, `1`=明天, `2`=后天 |
| `--time-slot` | 时间段 | ✅ | `14:00-15:00` / `19:00-20:00` |
| `--dry-run` | 干跑模式（不占名额） | 可选 | 加在任意位置 |

## 推荐工作流（3步确认法）

### Step 1：dry-run 预览可用性

```bash
cd "$SZU_BOOKING_DIR" || cd .
python main.py \
  --dry-run \
  --username 2023150090 \
  --password 你的密码 \
  --campus 粤海校区 \
  --sport 网球 \
  --date 1 \
  --time-slot 14:00-15:00
```

**输出解读**：
```
可用时间段 (4/14 个):
  [6] [OK] 14:00-15:00(可预约)    ← 可用，可以预约
  [11] [X] 19:00-20:00(已被预约)  ← 已被抢，无法预约
```

### Step 2：向用户确认意图

展示 dry-run 结果，明确询问：
- "14:00-15:00 时段显示可预约，要执行真实预约吗？"
- "19:00-20:00 已被预约，是否换其他时段？"

### Step 3：执行真实预约

获得用户明确确认后，去掉 `--dry-run` 执行：

```bash
cd "$SZU_BOOKING_DIR" || cd .
python main.py \
  --username 2023150090 \
  --password 你的密码 \
  --campus 粤海校区 \
  --sport 网球 \
  --date 1 \
  --time-slot 14:00-15:00
```

**成功标志**：输出中包含 `[OK] 预约成功（页面确认）`

## 错误处理

| 错误关键词 | 原因 | 处理方式 |
|-----------|------|---------|
| `选项不可用` | 时间段已被抢 | 建议换其他时段，先 dry-run 查可用时段 |
| `UnicodeEncodeError` | Windows GBK 终端问题 | 代码已使用 `[OK]`/`[X]` 替代，无需处理 |
| `未找到元素` | 页面结构变化 | 告知用户需要更新选择器代码 |
| `登录失败` | 账号密码错误 | 引导用户检查凭证 |
| `ReadTimeout` | 网络超时 | 重试一次，或检查网络连接 |
| `ECONNREFUSED` | Gateway 未运行 | 用户需先启动 Gateway |

## 注意事项

1. **时间敏感**：预约系统在特定时间开放（通常提前1-2天），建议提前测试流程
2. **并发限制**：同一账号同时只能有一个预约实例在运行
3. **密码安全**：
   - 禁止在对话中明文输出密码
   - 引导用户配置 `.env` 文件，运行时自动读取
   - 使用 `SZU_PASSWORD_0090=你的密码` 格式（学号后4位）
4. **dry-run 优先**：任何操作前先用 `--dry-run` 确认可用性
5. **首次配置**：必须设置 `SZU_BOOKING_DIR` 环境变量指向项目目录

## 成功标准

1. 输出包含 `[OK] 预约成功（页面确认）`
2. 日志目录 `logs\booking\runs\<date>_<trace-id>\` 中有成功记录

## 安全规则

- ✅ 先 `--dry-run` 预览，再向用户确认，最后执行真实预约
- ✅ 密码从 `.env` 文件读取，不在对话中暴露
- ❌ 不在没有任何预览的情况下直接执行真实预约
- ❌ 不在用户未明确确认的情况下执行真实预约
